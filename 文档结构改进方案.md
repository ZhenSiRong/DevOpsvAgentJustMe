## 二、核心问题诊断

### 问题 1：命名混乱，读者无法导航

| 现象 | 影响 |
|------|------|
| `AAA`、`AAB`、`AAC` 前缀编号 | 像临时草稿，读者不知道优先级或阅读顺序 |
| 中文目录名 + 英文文件名混用 |  inconsistent，增加认知负担 |
| `readme.md` vs `README.md` 大小写不统一 | 不规范 |
| `画给人看的图表` 语义模糊 | 不知道里面是什么 |

### 问题 2：内容重复 & 碎片化

| 现象 | 建议 |
|------|------|
| `AAB业务目标/原本.md` + `译本.md` | 合并为一份，保留最终版即可 |
| `AAG技术栈选型/` 按层拆 3 个目录 | 合并为一份表格式的技术栈选型文档 |
| `AAH架构设计/` 3 个空目录 + 1 个 eddx | 空目录删除，eddx 移到 docs/assets |
| `开发模式组合/` 2 个文档 | 合并为一份「开发规范」文档 |
| `AAC功能清单/scorePie.mmd` 仅 141B | 内容太少，合并到功能清单主文档 |

### 问题 3：缺乏统一入口 & 读者路径

**现状**：根目录没有 README，读者打开项目后面对 10 个目录无所适从。

**应该提供的路径**：
```
评委/评审老师 → 先看 README → 按顺序读 规划文档 → 看工程文档
新团队成员    → 先看 README → 按角色选择入口
你自己        → 比赛时快速定位需要展示的文档
```

## 三、改进方案：两层一入口

**核心思路**：
1. **根目录只保留一个 `README.md`** 作为总入口
2. **规划层** → 合并到 `docs-planning/` 目录
3. **工程层** → 保留在 `devops-agent/docs/` 目录
4. **每个目录一个 README** 作为该层入口
5. **统一命名规范**：全小写 + 连字符 + `.md`

### 改进后的目录结构


### 命名规范

| 规则 | 示例 |
|------|------|
| 全小写 | `Tech-Stack.md` → `tech-stack.md` |
| 连字符分隔 | `Backend_Function_Gap_Analysis.md` → `backend-gap-analysis.md` |
| 数字前缀表顺序 | `01-business-goals.md`、`02-functional-requirements.md` |
| README.md 作为目录入口 | 每个文档目录一个 `README.md` |

## 四、内容合并计划

### 合并 1：业务目标

**来源**：`AAB业务目标/原本.md` + `AAB业务目标/译本.md`
**目标**：`docs-planning/01-business-goals.md`
**策略**：以译本为底本，把原本中独有的内容（如果有）并入，删除重复叙述。

### 合并 2：技术栈选型

**来源**：`AAG技术栈选型/1.表现层/readme.md` + `2.业务逻辑层/readme.md` + `3.数据访问层/readme.md`
**目标**：`docs-planning/06-tech-stack.md`
**策略**：改为一张总表 + 每层一个小节，避免读者在三个文件间跳转。

### 合并 3：开发规范

**来源**：`开发模式组合/readme.md` + `TDD后端骨架+核心链路/readme.md`
**目标**：`docs-planning/08-development-guide.md`
**策略**：总览 + TDD 实践 + 核心链路设计，合并为一份。

### 合并 4：功能清单

**来源**：`AAC功能清单/baseNeed.mmd` + `scorePie.mmd`
**目标**：`docs-planning/02-functional-requirements.md`
**策略**：mindmap 保留在 `assets/` 作为配图，正文用 Markdown 表格重新组织（含评分权重列）。

### 迁移 5：架构设计

**来源**：`AAH架构设计/DevOpsAgent架构.eddx` + 空目录
**目标**：`assets/architecture-diagram.png` + `docs-planning/07-architecture.mmd`
**策略**：eddx 导出为 PNG/SVG 放入 assets，mindmap 用文字描述架构分层。

### 迁移 6：工程文档整理

**来源**：`devops-agent/docs/` 下 4 个文档
**目标**：统一小写命名，新增 `devops-agent/docs/README.md` 作为工程文档入口。
