# 磁盘空间清理（Disk Cleanup）

## 标准流程

### 第1步：分析现状
1. 使用 `disk_usage` 查看各分区使用率
2. 使用 `find_large_files --directory /var/log --min-size 100M` 找出大文件
3. 使用 `find_large_files --directory /tmp --min-size 50M` 检查临时文件

### 第2步：分类安全等级
- **安全删除**：日志轮转旧文件 (>7天)、软件包缓存、临时文件
- **需确认**：用户上传文件、备份文件
- **不可删除**：系统配置、数据库文件、运行时文件

### 第3步：清理建议
优先清理：
1. /var/log 中超过7天的旧日志（使用 logrotate 而非 rm）
2. dnf/yum 软件包缓存: `dnf clean all`
3. /tmp 中超过1天的文件

## 安全规则
- 绝不使用 `rm -rf /` 或 `rm -rf /*`
- 删除前先 `ls` 确认目标路径
- 系统目录 (/etc, /usr, /boot) 中的文件必须人工确认
- 每次操作后报告释放的空间

## 输出格式
向用户报告：
- 当前磁盘使用率
- 最大的5个文件/目录
- 可释放的预估空间
- 建议清理命令（需用户确认）
