# 12 安全体系说明

> 七层防护 + JWT 认证 + HITL 审批门 + 审计溯源

---

## 1. 整体架构

```
用户输入
   ↓
① 提示词注入检测 → 阻断高危输入
   ↓
② JWT 认证 → 拒绝未授权请求
   ↓
③ 命令安全校验 → 白名单/黑名单/路径保护
   ↓
④ DAG 编排安全 → 只读/写分类 + 管道安全检测
   ↓
⑤ Human-in-the-loop 审批门 → 危险操作人工确认
   ↓
⑥ 幂等控制 → 重复请求去重
   ↓
⑦ 审计日志 → 全链路溯源
```

---

## 2. 各层详解

### 2.1 提示词注入检测（`safety/prompt_injection.py`）

- 7 种攻击模式正则匹配（角色劫持、越狱、泄漏、越权、幻觉诱导等）
- 输出风险等级：safe / low / medium / high / critical
- critical 直接拦截并返回阻止消息

### 2.2 JWT 认证（`auth/auth.py` + `api/routes/auth.py`）

- 用户名密码登录 → JWT access token（24h 有效）
- 可选 JWT refresh token
- 所有 API 路由（除 `/auth/login`、`/health`、`/metrics`）均配置 `Depends(get_current_user)`
- 未认证请求返回 401

### 2.3 安全校验器（`safety/validator.py`）

| 规则 | 说明 |
|------|------|
| 危险路径 | 阻止写入 /etc、/boot、/var/lib/mysql 等系统目录 |
| 权限提升 | 阻止 `sudo`、`su`、`chmod 777` 等 |
| 进程占用 | 文件被进程占用时提示而非强制删除 |
| 黑名单命令 | `rm -rf /` 等破坏性命令直接拦截 |
| 白名单路径 | `config_guard.py` 维护可写路径白名单 |
| 批量校验 | 一次校验多条命令 |

### 2.4 DAG 安全分类（`orchestrator/planner.py`）

- 工具按类型分为 `READ_ONLY` / `WRITE` / `EXECUTE`
- `execute_command` 进一步分析命令内容
- 管道命令（`cmd1 | cmd2`）逐段安全检查
- 只读命令在 DAG 中并行执行，写入命令串行

### 2.5 Human-in-the-loop 审批门（`orchestrator/approval.py`）

**触发条件：** 命令匹配 10 种危险操作模式

```
rm -rf, systemctl stop/restart, chmod -R, chown -R,
写入 /etc//var//usr/ 等系统目录, mkfs, dd, fdisk, kill -9
```

**状态机：**

```
pending_approval → approved → 继续执行
                 → rejected → 跳过该操作
```

**API：**

```
POST /api/v1/orchestrator/runs/{run_id}/approve  {node_id, action: "approve"|"reject"}
GET  /api/v1/orchestrator/runs/{run_id}/pending
```

### 2.6 幂等控制（`safety/idempotency.py`）

- 基于请求签名的去重
- 防止重复提交导致的重复扣费/重复执行

### 2.7 审计日志

- `audit_logs` 表：每条操作（命令执行、工具调用）落审计
- 推理链路 `reasoning` 表：五段式 SENSE → ANALYZE → PLAN → EXECUTE → OUTPUT
- Prometheus metrics 指标暴露

---

## 3. 执行安全（`safety/executor.py`）

- 命令执行使用 `subprocess` + `shlex.quote()` 防注入
- 超时控制（默认 30s）
- 禁用 shell=True（避免 shell 注入）
- 用户级权限 `devops-runner` 运行

---

## 4. 配置写保护（`safety/config_guard.py`）

维护可写路径白名单，阻止对系统关键目录的意外写入操作。
