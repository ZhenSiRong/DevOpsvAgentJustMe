## 1. 开发测试环境

### 1.1 虚拟机基础信息

| 属性 | 值 |
|------|-----|
| IP 地址 | `192.168.187.128`（VMware NAT DHCP 分配，动态变化时需确认）|
| 操作系统 | 麒麟 V11 (Swan25) x86_64 |
| Python | 3.12.9（`/usr/local/python312/bin/python3`）|
| 连接方式 | SSH 密钥认证，root 用户，端口 22 |
| 项目路径（虚拟机内） | `/root/devops-agent/` |

### 1.2 SSH 执行规则

1. **每条命令只做一件事**：禁止链式 `pkill; sleep; start; check`
2. **必须加超时**：SSH/SCP 带 `-o ConnectTimeout=5`，curl 带 `--max-time 3`
3. **后台启动用 `setsid`**：`setsid python3 -m uvicorn ... > /tmp/log.log 2>&1 < /dev/null &`
4. **可能无输出的命令加 `|| true`** 兜底
5. **复杂内联 Python 用单引号包裹**

## 2. CI/CD 流水线

### 2.1 GitHub Actions

定义在 `.github/workflows/ci-cd.yml`，包含：

| Stage | 内容 |
|-------|------|
| lint | ruff 代码检查 |
| test | pytest 单元测试（包括 delivery 测试和 unit 测试）|
| build | 前端 npm build + 后端 wheels |
| deploy | SCP 至麒麟 VM + SSH 重启 |

### 2.2 手动部署

```bash
# 1. 本地构建前端
cd frontend && npm run build

# 2. 停止旧服务
ssh root@192.168.187.128 "pkill -f 'uvicorn devops_agent' || true"

# 3. 同步代码
scp -r src/devops_agent root@192.168.187.128:/root/devops-agent/src/
scp -r frontend/dist      root@192.168.187.128:/root/devops-agent/frontend/

# 4. 启动新服务（setsid 确保终端断开后进程不退出）
ssh root@192.168.187.128 "cd /root/devops-agent && setsid python3 -m uvicorn devops_agent.main:app --host 0.0.0.0 --port 8000 > /tmp/devops.log 2>&1 < /dev/null &"

# 5. 验证
curl http://192.168.187.128:8000/health
```

## 3. 服务管理

### 3.1 启动

```bash
cd /root/devops-agent
python3 -m uvicorn devops_agent.main:app --host 0.0.0.0 --port 8000
```

### 3.2 健康检查

```bash
curl http://192.168.187.128:8000/health
# {"status":"ok","app":"DevOps-Agent","version":"0.1.0","db_connected":true,"uptime_seconds":...}
```

### 3.3 查看日志

```bash
tail -f /tmp/devops_agent.log   # 服务日志
```

## 4. 测试

```bash
# 单元测试（纯 Python，无需服务）
cd /root/devops-agent
python3 -m pytest tests/unit/ -v

# 交付测试（需服务运行）
python3 -m pytest tests/delivery/ -v

# 全部
python3 -m pytest tests/unit/ tests/ -v
```
