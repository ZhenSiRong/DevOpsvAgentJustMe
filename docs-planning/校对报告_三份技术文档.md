## 一、05-api-contract.md

### P0 问题

#### 1. 聊天接口拆分描述错误

- **文档描述**：POST /api/v1/chat 带 stream: true/false 参数，一个端点两用
- **实际代码**：拆成两个独立端点
  - POST /api/v1/chat — 非流式，返回 APIResponse 包装 JSON
  - POST /api/v1/chat/stream — SSE 流式，返回 StreamingResponse
- **影响**：前端按照这个契约写会直接 404

#### 2. 响应格式严重偏差（几乎所有端点）

- **文档描述**：示例响应都是裸对象
- **实际代码**：后端几乎全部路由返回统一包装格式 APIResponse
  ```json
  {"status": "success", "data": {...}, "message": "OK"}
  ```
- **影响端点**：probe、chat、sessions、audit-logs、config、tools、safety 等全部端点

#### 3. 大量接口缺失（文档只列约 7 个，实际 50+）

| 模块 | 缺失端点 |
|------|---------|
| 健康检查 | GET /api/v1/health 及 /health/db/probe/agent/all |
| 配置管理 | GET/PUT/DELETE/POST /api/v1/config/* 全套 6 个端点 |
| 安全层 | GET/POST/PUT /api/v1/safety/* 全套 8 个端点 |
| 执行引擎 | POST /api/v1/execute |
| 推理链路 | GET /api/v1/reasoning/{session_id} 及 /summary |
| 工具管理 | GET/POST/PUT/DELETE /api/v1/tools/* 全套 6 个端点 |
| 审计日志 | GET /api/v1/audit/stats |

#### 4. WebSocket 接口未实现

- **文档描述**：WS /api/v1/ws/chat 推送五种消息
- **实际代码**：全局搜索无 WebSocket 路由实现，只有 SSE 流式
- **建议**：删除或标注"规划中，未实现"

#### 5. 认证接口未实现

- **文档描述**：POST /api/v1/auth/login、register、refresh
- **实际代码**：无 auth 路由模块
- **建议**：删除或标注"规划中，未实现"

#### 6. 错误格式描述错误

- **文档描述**：{"error": {"code": "...", "message": "...", "details": {...}}}
- **实际代码**：FastAPI HTTPException 返回 {"detail": "..."}；业务错误返回 {"status": "error", "message": "...", "data": null}

### P1 问题

#### 7. 探针查询参数不完整

- 文档：?path=/var/log&top_n=10
- 实际：还有 pattern、unit、hours 等参数

#### 8. 审计日志端点不完整

- 文档只列了 GET /api/v1/audit-logs
- 实际还有 GET /api/v1/audit/stats

### P0 问题

#### 1. 标题与文件名不符

- 文件名：08-development-guide.md
- 文档标题：08 开发规范
- 建议：统一为"开发指南"或"开发规范"

#### 2. 所有文件路径全部过时

| 文档中的路径 | 实际路径 | 问题 |
|-------------|---------|------|

#### 3. 路由描述严重过时

- 实际：10 个路由模块、50+ 端点

## 三、architecture-decisions.md

### P0 问题

#### 1. 标题与文件定位完全不符

- 文档标题：Brainstorm：用户动态添加 MCP Tool 方案
- 文件名：architecture-decisions.md
- 问题：这不是架构决策记录（ADR），是一份设计 brainstorm。缺少决策背景、方案对比、选择理由、后果分析。
- 建议：重命名为 dynamic-tool-design.md，或重写为真正的 ADR

#### 2. API 设计与实际实现不符

| 文档描述 | 实际代码 | 偏差 |
|---------|---------|------|
| GET /api/v1/tools 返回 {"builtin": [...], "dynamic": [...]} | 返回 {"items": [...], "total": N} | 不区分 builtin/dynamic |
| POST /api/v1/tools/check_service/test | POST /api/v1/tools/{id}/test | 路径参数是 id:int，不是 name |
| 请求体字段 schema | 实际模型字段 parameters，DB 字段 schema_json | 命名不一致 |

### P1 问题

#### 3. 缺少实现状态标注

- 文档 P1/P2/P3/P4 是计划，但未标注哪些已实现
- 从代码验证：P1 已完成，P2 HTTP 执行器模型层已支持，但文档未反映
