# 01 - 项目记忆 (Project Memory)

> **父文档**: [Coding Agent 上下文体系](./Coding-Agent-Context-Taxonomy.md)  
> **定义**: 项目在生命周期中积累的持久化知识资产

---

## 1.1 文档目的

项目记忆是确保 Agent 行为一致性和知识传承的核心机制。它记录了：
- 为什么要做某个技术决策
- 曾经踩过什么坑
- 项目的演进历史

---

## 1.2 组成部分

### 1.2.1 决策历史 (Decision Log)

| ID | 决策主题 | 选择 | 替代方案 | 决策日期 | 决策者 |

| ADR-001 | 数据库选型 | PostgreSQL | MySQL/MongoDB | 2026-01-15 | @arch-team |

---

## 1.4 维护规则

| 规则 | 说明 |
|------|------|
| 新增时机 | 每个技术决策后立即记录 |
| 更新时机 | 发现新问题时追加到问题库 |
| 审核频率 | 每季度审查一次决策有效性 |
| 清理规则 | 超过2年的过时决策归档 |

## 1.5 与其他上下文的关系

```

    │
    ├──→ 影响: 项目规则 (基于历史决策)
    │
    ├──→ 支持: 技术上下文 (解释为什么选择现有技术)
    │
    └──→ 依赖: 对话上下文 (用户可询问历史决策)
```
