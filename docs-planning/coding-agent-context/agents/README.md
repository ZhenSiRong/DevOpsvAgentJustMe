## 角色职责速查

### 🎯 核心角色（直接产出代码）

| 角色 | 主要产出 | 协作对象 |
|------|----------|----------|
| 前端工程师 | React 组件、页面 | PM、后端、QA |
| 后端工程师 | API、业务逻辑 | PM、前端、QA |
| 测试工程师 | 测试用例、bug 报告 | 开发、Reviewer |
| 代码审查 | Review 意见 | 开发、PM |

### 🔧 支持角色（保障流程）

| 角色 | 主要产出 | 协作对象 |
|------|----------|----------|
| PM | 任务分配、进度报告 | 所有 Agent |
| Git Manager | 分支规范、发布 | 开发、DevOps |
| 技术文档 | API 文档、指南 | 开发、用户 |

### 🏢 特殊角色

| 角色 | 双重身份 | 特殊定位 |
|------|----------|----------|
| 运维工程师 | 需求方 + 运维 | 核心用户、验收者 |

## 设计原则

### 1. 角色隔离
- 每个 Agent 只处理其职责范围内的事务
- 跨角色协作通过消息传递（PM 协调）

### 2. 上下文边界
- Agent 只能访问其角色所需的上下文维度
- 敏感信息按需授权

### 3. 能力对等
- 工具权限与职责匹配
- 禁止越权操作

### 4. 双向通信
- 开发 Agent → PM → 需求方
- 需求方 → PM → 开发 Agent

---

## Hermes 结构

每个 Agent 的 Hermes 包含以下部分：

```markdown
1. 角色定义          # 身份定位
2. 核心职责          # 主要任务
3. 工作原则          # 行为准则
4. 上下文权限        # 可访问的上下文
5. 工具能力          # 可用的工具
6. 禁止行为          # 红线
7. 成功标准          # 衡量指标
8. 协作规范          # 与其他 Agent 的交互
```
