# 04 - 业务上下文 (Business Context)

> **父文档**: [Coding Agent 上下文体系](../Coding-Agent-Context-Taxonomy.md)  
> **定义**: 项目的业务领域知识、目标和约束

---

## 4.1 文档目的

业务上下文帮助 Agent 理解 "为什么要做这个功能" 而非仅仅 "如何实现"，确保技术决策与业务目标一致。

---

## 4.2 组成部分

### 4.2.1 领域知识 (Domain Knowledge)

#### 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 用户 | User | 系统的使用者，可以是管理员或普通成员 |
| 租户 | Tenant | 组织级别的隔离单元 |
| 角色 | Role | 定义权限集合 |
| 资源 | Resource | 系统管理的对象 |

#### 业务规则

```markdown
## 权限规则

1. 超级管理员可以管理所有租户
2. 租户管理员只能管理本租户内的用户
3. 普通用户只能访问自己创建的资源
4. 权限变更立即生效

## 数据流转规则

1. 用户创建的资源默认私有
2. 资源可以被显式分享
3. 删除用户时，其资源由管理员决定保留/删除
```

### 4.2.2 功能规格 (Feature Specifications)

#### 功能矩阵

| 功能模块 | 功能点 | 优先级 | 状态 |
|----------|--------|--------|------|
| 用户管理 | 用户注册 | P0 | 已上线 |
| 用户管理 | 用户登录 | P0 | 已上线 |
| 权限管理 | 角色配置 | P1 | 开发中 |
| 权限管理 | 权限分配 | P1 | 待开发 |
| 资源管理 | 资源创建 | P0 | 已上线 |
| 资源管理 | 资源分享 | P2 | 规划中 |

#### 用户故事

```markdown
## 用户故事模板

作为 [角色],
我希望 [功能],
以便 [收益].

验收标准:
- Given [前置条件]
- When [操作]
- Then [预期结果]
```

**示例**:
```markdown
作为 租户管理员,
我希望 能够批量导入用户,
以便 快速完成团队初始化.

验收标准:
- Given 我已登录且有管理员权限
- When 我上传包含 100 个用户的 CSV 文件
- Then 系统应在 30 秒内完成导入并显示结果
```

### 4.2.3 业务流程 (Business Processes)

#### 核心流程

```
[用户注册] → [邮箱验证] → [创建租户] → [初始化配置]
                                    ↓
                              [邀请成员]
                                    ↓
                         ┌──────────┴──────────┐
                         ↓                     ↓
                  [普通用户流程]          [管理员流程]
                         ↓                     ↓
                  [使用功能]             [管理配置]
```

#### 状态机定义

```markdown
## 资源状态机

[草稿] ──发布──→ [已发布] ──归档──→ [已归档]
   │                   ↑
   └──删除──→ [已删除] ─┘
```

### 4.2.4 约束条件 (Constraints)

| 类型 | 约束 | 说明 |
|------|------|------|
| 法规 | GDPR 合规 | 用户数据可删除、导出 |
| 性能 | API 响应 < 200ms | P95 延迟 |
| 可用性 | SLA 99.9% | 每月停机 < 44 分钟 |
| 容量 | 单租户支持 10,000 用户 | 可扩展 |
| 安全 | 通过 SOC 2 审计 | 年度审计 |

## 4.4 优先级定义

| 优先级 | 标识 | 定义 | 交付要求 |
|--------|------|------|----------|
| P0 | 紧急 | 核心业务，中断流程 | 立即处理 |
| P1 | 高 | 重要功能，影响效率 | 1 周内 |
| P2 | 中 | 辅助功能，体验优化 | 2 周内 |
| P3 | 低 | 锦上添花 | 排期待定 |
