# 05 - 对话上下文 (Conversation Context)

> **父文档**: [Coding Agent 上下文体系](../Coding-Agent-Context-Taxonomy.md)  
> **定义**: 当前对话的会话状态和历史信息

---

## 5.1 文档目的

对话上下文确保 Agent 在多轮交互中保持连贯性，理解用户的真实意图，并记住之前的承诺和讨论。

---

## 5.2 组成部分

### 5.2.1 当前任务状态 (Current Task State)

```markdown
## 任务卡片

**任务 ID**: TASK-2026-0429-001
**任务类型**: 功能开发 / Bug修复 / 重构 / 咨询
**创建时间**: 2026-04-29 17:30

### 目标
{用户期望的最终结果，清晰描述}

### 进度
- [x] 步骤1: 已完成
- [ ] 步骤2: 进行中
- [ ] 步骤3: 待处理

### 阻塞点
- 问题: {当前遇到的具体问题}
- 已尝试: {尝试过的解决方案}
- 需要的帮助: {需要什么资源或决策}
```

### 5.2.2 历史对话摘要 (Conversation History Summary)

#### 摘要模板

```markdown
## 对话摘要 #{n}

**用户**: {主要需求和意图}
**Agent**: {已提供的解决方案}
**结果**: {当前状态}

### 关键决定
- {决定1}: {详情}
- {决定2}: {详情}

### 待确认
- [ ] {问题1}: {描述}
- [ ] {问题2}: {描述}

### 上下文窗口
{最近 5-10 轮对话的核心内容提炼}
```

#### 用户偏好记录

| 偏好类型 | 设置 |
|----------|------|
| 代码风格 | 详细注释 / 简洁优先 |
| 解释深度 | 技术细节 / 概要说明 |
| 响应格式 | 分点说明 / 连续段落 |
| 语言 | 中文 / 英文 |

### 5.2.3 已讨论内容 (Discussed Content)

```markdown
## 已明确的需求

1. 使用 PostgreSQL 作为主数据库 ✅
2. 鉴权使用 JWT Token ✅
3. 拒绝使用 JavaScript，优先 TypeScript ✅

## 已否决的方案

1. ~~MongoDB~~ → 原因: 需要事务支持
2. ~~Session 认证~~ → 原因: 分布式不友好

## 已确认的假设

- 用户环境: Linux/macOS
- 部署方式: Docker 容器化
- 团队规模: 5-10 人
```

### 5.2.4 上下文窗口管理 (Context Window Management)

```markdown
## 上下文窗口策略

### 保留内容 (高优先级)
- 当前任务的完整描述
- 最近修改的文件列表
- 用户明确提出的约束
- 未解决的问题

### 可丢弃内容 (低优先级)
- 已完成步骤的详细日志
- 已确认的技术选型讨论
- 无关的闲聊内容

### 压缩策略
当上下文超过阈值时:
1. 压缩历史对话为摘要
2. 保留最新的 N 轮完整对话
3. 移除已完成的子任务详情
```

---

## 5.4 上下文边界规则

| 规则 | 说明 |
|------|------|
| 任务边界 | 一个任务专注于一个目标 |
| 上下文污染 | 不将上一个任务的状态带入新任务 |
| 假设声明 | 任何假设必须明确声明 |
| 澄清优先 | 歧义时优先询问而非猜测 |

---

## 5.5 与其他上下文的关系

```
对话上下文
    │
    ├──→ 依赖: 项目记忆 (查询历史决策)
    │
    ├──→ 应用: 项目规则 (执行代码规范)
    │
    ├──→ 驱动: 技术上下文 (使用技术栈)
    │
    └──→ 服务: 业务上下文 (实现业务目标)
```

## 5.6 最佳实践

### 5.6.1 开始新任务时的检查清单

- [ ] 确认任务目标是否清晰
- [ ] 检查项目规则是否相关
- [ ] 回顾类似任务的历史经验
- [ ] 识别需要澄清的问题
