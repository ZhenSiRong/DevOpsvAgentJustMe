## 2. 路由结构

```jsx
<Route path="/">            // Layout + PrivateRoute
  <Route index              → ChatPage       // Agent 对话
  <Route path="probe"       → ProbePage      // OS 探针
  <Route path="audit"       → AuditPage      // 审计日志
  <Route path="safety"      → SafetyPage     // 安全中心
  <Route path="reasoning"   → ReasoningPage  // 推理链路
  <Route path="orchestrator"→ OrchestratorPage // 任务编排
  <Route path="mcp"         → MCPPage        // MCP 管理
  <Route path="evolution"   → EvolutionPage  // 自演进（新增）
  <Route path="skills"      → SkillsPage     // Skills（新增）
  <Route path="settings"    → SettingsPage   // 系统设置
```

**共 11 个路由页面**，全部在侧边栏有对应菜单项。

---

## 3. 侧边栏导航

定义在 `Layout.jsx` 的 `navItems` 数组，包含 10 个菜单：

```
Agent 对话 → OS 探针 → 安全中心 → 审计日志 → 推理链路
→ 任务编排 → MCP 管理 → 自演进 → Skills → 系统设置
```

支持折叠（窄模式 64px）和移动端抽屉。

---

## 4. 核心页面

### 4.1 ChatPage — Agent 对话

- SSE 流式消息渲染（`streamChatFetch`）
- 五段式推理链路实时动画
- 侧面板：终端输出 + 推理链路流程图 + DAG 可视化
- 消息重试（`retryMessage`）

### 4.2 SettingsPage — 系统设置

4 个 Tab：

| Tab | 内容 |
|-----|------|
| LLM 配置 | 协议/Base URL/API Key/温度/最大 Token |
| 模型路由 | 模型池状态（健康/熔断/失败率）+ 重置按钮 |

### 4.3 EvolutionPage — 自演进

- 统计卡片（经验教训 / 事实知识 / 平均重要性）
- 经验教训列表
- 人类专家反馈表单

### 4.4 SkillsPage — Agent Skills

- Skills 目录结构展示
- 已加载 Skill 列表
- 添加新 Skill 指南

## 5. API 通信

| 方式 | 用途 | 实现 |
|------|------|------|
| `request()` | 常规 REST | 自动附加 JWT Token，处理 401 自动清除 |
| `streamChatFetch()` | SSE 流式聊天 | `fetch` + `ReadableStream`，支持自动重连（指数退避）|
| 直接 `fetch` | Settings 新面板 | 读取 `localStorage('auth_token')` 手动加头 |

---

## 6. 认证流程

```
LoginPage → POST /api/v1/auth/login → 返回 JWT
  → 存入 localStorage('auth_token')
  → AuthContext 提供全局 user 状态
  → PrivateRoute 检查未登录则重定向到 /login
```

Token 有效期 24h，过期后自动清除并返回登录页。
